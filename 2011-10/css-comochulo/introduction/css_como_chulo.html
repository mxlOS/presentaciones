¿Qué es less?

oficialmente:

LESS extiende a CSS con comportamiento dinámico como variables, mixins, operaciones y funciones. LESS funciona en el lado del cliente (IE 6+, Webkit, Firefox) y del lado del servidor, con Node.js.

qué significa esto?

Si estamos acostumbrados a usar variables, métodos y funciones en un lenguaje de programación (PHP, Ruby, Perl, Python, etc.), ¿por qué no tratar el CSS (que no es un lenguaje de programación) como tal?

¿Qué ganamos?

Facilidad para escribir CSS, ya que es más facil declarar variables, métodos y funciones y REUTILIZARLOS cuando los necesitemos, a tener que escribir todo de nuevo.

Ejemplos sencillos

Variables

// LESS
@color: #4D926F;
#header {
color: @color;
}
h2 {
color: @color;
}
/* convertido a CSS */
#header {
color: #4D926F;
}
h2 {
color: #4D926F;
}

Mixins

// LESS
.rounded-corners (@radius: 5px) {
border-radius: @radius;
-webkit-border-radius: @radius;
-moz-border-radius: @radius;
}

#header {
.rounded-corners;
}
#footer {
.rounded-corners(10px);
}
/* convertido a CSS */
#header {
border-radius: 5px;
-webkit-border-radius: 5px;
-moz-border-radius: 5px;
}
#footer {
border-radius: 10px;
-webkit-border-radius: 10px;
-moz-border-radius: 10px;
}

Nesting

// LESS
#header {
h1 {
font-size: 26px;
font-weight: bold;
}
p { font-size: 12px;
a { text-decoration: none;
&:hover { border-width: 1px }
}
}
}
/* convertido a CSS */
#header h1 {
font-size: 26px;
font-weight: bold;
}
#header p {
font-size: 12px;
}
#header p a {
text-decoration: none;
}
#header p a:hover {
border-width: 1px;
}
